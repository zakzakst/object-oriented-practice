# 実装用の図

## ユースケース図
```puml
@startuml
left to right direction
actor "ユーザー" as user
rectangle アプリ {
  usecase "メモを管理する" as case1
}
user -- case1
@enduml
```


## オブジェクト図
```puml
@startuml
object ユーザー1
ユーザー1 : ID = ユーザーID1

object メモ1
メモ1 : タイトル = 今日の朝ごはん
メモ1 : 内容 = 米、みそ汁、卵焼き

object メモ2
メモ2 : タイトル = 買ってくるもの
メモ2 : 内容 = 味噌、卵3個

object メモ3
メモ3 : タイトル = 朝の予定
メモ3 : 内容 = 顔を洗う、ごはんを食べる、歯磨き

ユーザー1 -- メモ1
ユーザー1 -- メモ2
ユーザー1 -- メモ3
@enduml
```


## クラス図
```puml
@startuml
class ユーザー {
  - ID : 文字列
}
class メモ {
  - タイトル : 文字列
  - 内容 : 文字列
  - ユーザーID : 文字列
  + メモを管理する() : void
}
ユーザー "1" -- "0.. *" メモ : 保持する
@enduml
```


## シーケンス図
```puml
@startuml
box メモを管理する
ユーザー -> firebase: メモの管理
note right : データベースの処理は\nfirebase側で行う
end box
@enduml
```


## ER図
```puml
@startuml
entity ユーザー {
  + ID [PK]
}
entity メモ {
  + ID [PK]
  ---
  タイトル
  内容
  (ユーザーID)
}
note right: ユーザーIDは\nデータベースの\nパスで紐づける
ユーザー --o{ メモ
@enduml
```
