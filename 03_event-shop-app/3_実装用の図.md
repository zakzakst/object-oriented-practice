## クラス図
```puml
@startuml
class 会場 {
  - ID : 文字列
  - 名称 : 文字列
  - 写真 : 文字列
  + 会場を管理する() : void
}
class 店舗 {
  - ID : 文字列
  - 名称 : 文字列
  - 電話番号 : 文字列
  - メニュー : 文字列
  - 紹介文 : 文字列
  - 写真 : 文字列
  - 会場名 : 文字列
  - 場所 : 文字列
  + 店舗を管理する() : void
  + 会場名・場所を取得する() : void
}
class 店舗表 {
  - 会場ID : 文字列
  - 店舗ID : 文字列
  - 場所 : 文字列
  + 店舗の場所を管理する() : void
}
会場 "1" -- "0.. *" 店舗 : 所属する
@enduml
```


## シーケンス図
```puml
@startuml
box 会場・店舗を管理する
participant ユーザー
participant Nuxt
participant Firebase as FB
participant FireStorage as FS

ユーザー -> Nuxt : データ入力

group データ
Nuxt -> FB : データベース更新
note over FB : データベースの処理
FB -> Nuxt : データベース更新完了
end

group 写真
Nuxt -> FS : ストレージ更新
note over FS : ストレージの処理
FS -> Nuxt : ストレージ更新完了
end

Nuxt -> ユーザー : 画面更新
end box
@enduml
```

```puml
@startuml
box 店舗の会場名・場所を取得する
participant ユーザー
participant Nuxt
participant Firebase as FB

ユーザー -> Nuxt : 店舗ID

group 店舗表データ
Nuxt -> FB : 店舗表の\nコレクションデータ\nリクエスト
note over FB : データベースの処理
FB -> Nuxt : 店舗表の\nコレクションデータ
end

note over Nuxt : 店舗表と店舗IDを照合し、\n対象の会場ID・場所を取得

group 会場データ
Nuxt -> FB : 会場の\nシングルデータ\nリクエスト
note over FB : データベースの処理
FB -> Nuxt : 会場の\nシングルデータ
end

Nuxt -> ユーザー : 画面更新
end box
@enduml
```
